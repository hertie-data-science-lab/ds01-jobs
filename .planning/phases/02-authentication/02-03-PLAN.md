<task type="auto">
  <name>Task 1: Create Typer CLI with all key management commands</name>
  <files>src/ds01_jobs/cli.py</files>
  <action>
  Create src/ds01_jobs/cli.py with a Typer app implementing 4 commands:

  **Helper functions:**

  1. `generate_api_key() -> tuple[str, str]`: Generate `ds01_` + 32 bytes base64url. Return (raw_key, key_id) where key_id is first 8 chars of the base64url portion. Use `secrets.token_bytes(32)` and `base64.urlsafe_b64encode(...).rstrip(b"=").decode()`.

  2. `parse_duration(duration: str) -> int`: Parse duration strings like "90d", "30d", "180d" into days (int). Only need to support `Nd` format. Raise `typer.BadParameter` on invalid format.

  3. `check_org_membership(username: str, org: str) -> bool`: Check GitHub org membership. Read `GITHUB_TOKEN` from env. If present, use authenticated endpoint (`/orgs/{org}/members/{username}` with Bearer token). If absent, use public endpoint (`/orgs/{org}/public_members/{username}`). Return True if status 204, False if 404. Use `httpx.get()` with 10s timeout. On network error, print error and exit (per locked decision: live API call only, no fallback).

  **Commands:**

  4. `key_create(username: str, expires: str = "90d", json_output: bool = False)`:
     - Check GitHub org membership. If not a member, print error and exit 1.
     - Check if user already has an active (non-revoked, non-expired) key. If yes, print error: "User {username} already has an active key. Use key-revoke or key-rotate first." and exit 1.
     - Generate key via `generate_api_key()`.
     - Hash with `bcrypt.hashpw(raw_key.encode(), bcrypt.gensalt(rounds=12))`.
     - Parse duration, compute expires_at = now + timedelta(days=N).
     - INSERT into api_keys (username, key_id, key_hash, created_at, expires_at).
     - If `--json`: print JSON with username, key (raw), key_id, expires_at.
     - If plain text: print the raw key prominently, then metadata (username, expiry date), then a copy-pasteable setup instructions block:
       ```
       API Key created successfully

       Key:     ds01_ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmno

       User:    researcher1
       Expires: 2026-06-04

       Setup instructions (send to researcher):
       ─────────────────────────────────────────
       mkdir -p ~/.config/ds01
       echo "DS01_API_KEY=ds01_ABCDEF..." > ~/.config/ds01/credentials
       ─────────────────────────────────────────
       ```
     - Use plain print() for output (gh/stripe style, no Rich decorators).

  5. `key_list(json_output: bool = False)`:
     - SELECT all from api_keys (including revoked).
     - Compute status for each: "active" (not revoked, not expired), "revoked", "expired".
     - If `--json`: print JSON array of all keys with fields.
     - If plain text: print aligned columns - USERNAME, STATUS, CREATED, EXPIRES, LAST USED.
       Format dates as YYYY-MM-DD. last_used shows "never" if NULL.

  6. `key_revoke(username: str, yes: bool = False, json_output: bool = False)`:
     - Look up user's key. If not found or already revoked, print error and exit 1.
     - If not `--yes`: `typer.confirm(f"Revoke key for {username}?", abort=True)`
     - UPDATE api_keys SET revoked = 1 WHERE username = ?.
     - If `--json`: print JSON with username, status: "revoked".
     - If plain text: print "Key revoked for {username}".

  7. `key_rotate(username: str, expires: str = "90d", yes: bool = False, json_output: bool = False)`:
     - Look up user's existing active key. If no active key, print error and exit 1.
     - If not `--yes`: `typer.confirm(f"Rotate key for {username}?", abort=True)`
     - In a single transaction: revoke old key (UPDATE SET revoked=1), create new key (INSERT with new key_id, key_hash, dates).
     - Note: UNIQUE constraint on username means we must DELETE the old row and INSERT, OR update the existing row with new key data. Since we want revocation history, UPDATE the existing row to revoked=1 and change username to `{username}_revoked_{timestamp}` to free the UNIQUE constraint, then INSERT the new key. Alternative (simpler): just UPDATE the existing row with new key_id, key_hash, created_at, expires_at, revoked=0, last_used_at=NULL. This loses revocation history but keeps it simple. Use the UPDATE approach (simpler, single-key-per-user means history isn't needed).
     - Display new key same as key-create output.

  **Typer app setup:**
  ```python
  app = typer.Typer(
      name="ds01-job-admin",
      help="DS01 Job Submission Service - Admin CLI",
  )
  ```

  All `--json` flags: use `Annotated[bool, typer.Option("--json", help="JSON output")]` with default False.
  All `--yes` flags: use `Annotated[bool, typer.Option("--yes", "-y", help="Skip confirmation")]` with default False.
  All `--expires` flags: use `Annotated[str, typer.Option(help="Key validity duration (e.g. 90d, 30d, 180d)")]` with default "90d".
